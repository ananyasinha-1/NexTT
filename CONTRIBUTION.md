# Contributing to NexTT

Thank you for your interest in contributing to Smart Campus! Contributions of all sizes are welcome.

## Ways to Contribute

* Fix bugs and improve performance.
* Add new features to the React frontend or Node.js backend.
* Improve UI/UX and accessibility.
* Enhance documentation.
* Write tests and improve code quality.
* Report bugs and suggest improvements.

## Getting Started

### 1. Fork the Repository

Create your own fork of the project.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/smart-campus.git
cd smart-campus
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

## Development Guidelines

* Follow the existing code style and project structure.
* Write clean, maintainable, and documented code.
* Keep pull requests focused on a single issue.
* Update documentation when necessary.
* Test your changes before submitting.

## Pull Request Process

Before opening a Pull Request:

* Ensure your changes work as expected.
* Perform a self-review.
* Include screenshots for UI-related changes.
* Reference the related issue in your PR description.

### Example PR Titles

```text
feat: add student attendance dashboard
fix: resolve login authentication issue
docs: update installation guide
ui: improve mobile responsiveness
```

## Issue Reporting

When creating an issue:

* Use a clear title.
* Provide detailed steps to reproduce bugs.
* Include screenshots when applicable.

Thank you for contributing to NexTT:Smart Campus!
